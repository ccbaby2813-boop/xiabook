## 🔍 问题 1：为什么有两个 node 进程？

### 现象
```bash
ps aux | grep "node.*server.js"
# 显示 2 个进程
```

### 根因分析

**原因 1：手动重复启动**
- 每次执行 `nohup node src/server.js &` 都会启动新进程
- 旧进程未被完全停止
- 导致多个进程同时运行

**原因 2：没有进程管理器**
- 未使用 PM2 等进程管理工具
- 没有自动检测和防止重复启动的机制
- 进程崩溃后不会自动重启

**原因 3：端口占用错误被忽略**
```
Error: listen EADDRINUSE: address already in use 0.0.0.0:3000
```
- 第一个进程已占用 3000 端口
- 第二个进程启动失败但仍存在
- 导致多个进程竞争

### 进程关系

| 进程 | PID | 启动方式 | 状态 |
|------|-----|----------|------|
| server.js | 452806 | nohup | 运行中 |
| brain/index.js | 379421 | Brain Scheduler | 运行中 |

**结论**：两个进程是独立的，不是主从关系

## 🔍 问题 2：为什么老板看到"加载中"？

### 现象
- 所有板块显示"加载中"
- 帖子卡片不显示
- 前端一直等待数据

### 根因分析

**可能原因 1：前端 JS 错误**
```javascript
// app.js loadPosts 函数
const { data } = await res.json();
renderPostGrid(data || [], false);
```
- 如果 API 返回格式变化，可能导致解析失败
- catch 块捕获错误但只显示"加载失败"

**可能原因 2：浏览器缓存**
- 缓存了旧版本 app.js
- 旧代码调用新 API 失败
- 导致数据加载失败

**可能原因 3：API 响应格式**
- API 返回 `{success: true, data: [...]}`
- 前端期望 `{data: [...]}`
- 导致解析错误

### 验证结果

**API 测试**：
```bash
curl https://xiabook.cn/api/posts?limit=1
# 返回正常 ✅
```

**前端代码检查**：
```javascript
const { data } = await res.json();
// 正确解构 {success: true, data: [...]}
```

**结论**：API 正常，前端代码正常，问题可能是浏览器缓存

## 🔍 问题 3：缓存模块缺失

### 现象
- API 代码调用 `cache.get()`
- 但 `src/utils/cache.js` 文件不存在
- 导致服务器启动失败或异常

### 根因
- 缓存模块被误删或未创建
- 代码引用了不存在的模块
- 导致未捕获异常

### 修复
- 创建 `src/utils/cache.js`
- 实现简单的内存缓存
- 服务器正常启动 ✅

## 📋 解决方案

### 1. 使用 PM2 管理进程

```bash
# 安装 PM2
npm install -g pm2

# 启动服务器
pm2 start src/server.js --name xiabook

# 保存配置
pm2 save

# 开机自启
pm2 startup
```

**效果**：
- 防止重复启动
- 自动重启崩溃进程
- 统一管理日志

### 2. 添加进程锁

```bash
# 启动前检查
if ps aux | grep "node.*server.js" | grep -v grep > /dev/null; then
  echo "服务器已在运行"
  exit 1
fi
```

### 3. 前端缓存控制

```html
<!-- 添加版本号 -->
<script src="/js/app.js?v=1775137811"></script>
```

### 4. 完善错误处理

```javascript
try {
  const res = await fetch(url);
  const result = await res.json();
  if (!result.success) throw new Error(result.error);
  const { data } = result;
  renderPostGrid(data || [], false);
} catch(e) {
  console.error('[loadPosts] 错误:', e);
  grid.innerHTML = '<div class="error-tip">加载失败：' + e.message + '</div>';
}
```
