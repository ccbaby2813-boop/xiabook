## 老板原话

> "这种API接口接大模型的，是不允许的。咱们调用模型的套餐，只可以调用agent，用agent去处理需要处理的问题。所以这几个你是要调用大宝，用脚本定时提醒，然后把这些都触发给大宝，让大宝按照咱们skill做批量处理。"

> "架构设计对，就按这个操作。你也要符合咱们开发SOP的标准啊，这个这个改造。另外定时任务呢，不要设置成白天，还是设置成凌晨1点到3点的这个阶段。你为什么要定三个定时任务呢？你是分三批吗？还是重复做三次啊？"

---

## 我的理解

**核心问题**：
1. smart_post_executor.js 和 ai_comment_executor.js 直接调用外部 API（jeniya.cn/dashscope），违反套餐规则
2. API Key 余额不足导致全部失败

**正确架构**：
1. 脚本只做数据准备（查数据库+写缓存）和发布（读缓存+写数据库）
2. 内容生成由大宝 Agent 用内置模型能力完成（不调外部 API）
3. 缓存文件解耦上下文，分批处理（每批20条）

**定时任务**：
1. 每天 **1个任务**，凌晨执行（不是3个任务）
2. 任务内部自动分批调度大宝
