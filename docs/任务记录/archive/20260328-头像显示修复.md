## 我的理解

**问题分析**：
1. 首页右上角头像：如果用户头像是emoji（如🦞），直接赋值给`<img src="🦞">`会失败
2. profile-modal头像：同样的问题，emoji无法作为img.src显示
3. profile-modal和user.html设计风格不一致

**根本原因**：
- avatar可能是emoji（注册时选择的），也可能是URL（自定义上传的）
- 当前代码把所有avatar当作URL处理，直接赋值给img.src

## 技术方案

### 头像处理函数

```javascript
// 判断avatar类型并返回正确的HTML
function renderAvatarHtml(avatar, username) {
  if (!avatar) {
    // 无头像，显示首字母
    return `<div class="avatar-fallback">${(username||'?')[0]}</div>`;
  }
  
  // 判断是否为URL
  if (avatar.startsWith('http') || avatar.startsWith('/') || avatar.startsWith('data:')) {
    // URL类型，用img标签
    return `<img src="${avatar}" alt="" onerror="this.parentElement.innerHTML='<div class=\\'avatar-fallback\\'>${(username||'?')[0]}</div>'">`;
  }
  
  // emoji类型，直接显示
  return `<div class="avatar-emoji">${avatar}</div>`;
}
```
