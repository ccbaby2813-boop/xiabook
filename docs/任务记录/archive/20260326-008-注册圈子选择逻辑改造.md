### 2. 背景说明

**老板原话**：
> "只上10个领域，对应一个领域一个圈子，就是只给他十个圈子的选择空间。并且要提示他只能选择一个圈子加入。然后一旦这个圈子满50人了，这个圈子就从可选列表里下架。然后下一个注册用户就看不到这个选择了，从同领域的待命的圈子里补充一个进来，就永远保持用户是有10个可选择的圈子。"

**背景补充**：
- 当前后台有21个圈子，每个领域有active和reserve两种状态
- 需要实现动态的圈子选择机制
- 满50人自动下架+补充

### 3. 具体需求

- [ ] API改造：GET /circles?for_register=1 只返回10个active圈子
- [ ] 前端提示：显示"只能选择一个圈子加入"
- [ ] 注册流程：检查圈子是否满50人
- [ ] 满员下架：status从active改为full
- [ ] 自动补充：从同领域reserve圈子中补充一个

### 4. 技术方案

| 改动点 | 文件 | 说明 |
|--------|------|------|
| 圈子API | src/routes/api.js | for_register参数过滤 |

| 前端页面 | public/register.html | 提示文案+样式调整 |

### 5. 验收标准

1. 注册页面只显示10个圈子（每个领域1个）
2. 显示提示"只能选择一个圈子加入"
3. 注册后圈子满50人，自动下架
4. 同领域reserve圈子自动上架
