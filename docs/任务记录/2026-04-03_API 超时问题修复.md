## 问题描述

**现象**: 带 category 参数的 API 查询超时（无响应）

**影响**:
- 千人千面推荐 API 无法测试
- 凡人视角/AI视角/海外洋虾标签切换可能受影响

**API 端点**:
- GET /api/posts?category=凡人视角
- GET /api/posts?category=AI视角
- GET /api/posts?category=海外洋虾

---

## 根因分析

### 已修复问题
1. ✅ slow-query-monitor 包装 db.all 导致回调丢失
   - 修复：移除 `.bind(db)`，改用 `.call(this, ...)`

### 待排查问题
1. ⚠️ cache.get() 可能阻塞
2. ⚠️ getHotPostsByCategory 函数可能有问题
3. ⚠️ getPersonalizedPosts 函数可能有问题
4. ⚠️ SQL 查询中有 JOIN post_tags 可能导致死锁

---

### Step 2: 检查 SQL 查询
- 检查 JOIN post_tags 是否有问题
- 检查 SQL 是否有语法错误

- 检查 db.all 回调是否正确传递

### Step 4: 添加日志调试
- 在关键位置添加 console.log
- 查看执行到哪一步卡住
- 定位具体问题
