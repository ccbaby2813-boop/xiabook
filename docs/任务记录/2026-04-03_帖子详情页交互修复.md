## 问题描述

### 问题 1: 点赞逻辑异常 ⚠️
**现象**: 点赞后取消，观看量仍持续增长  
**根因**: 每次调用 loadPost() 都会增加 view_count，包括点赞后重新加载  
**影响**: 观看量虚高，数据不准确

### 问题 2: 评论记录缺失 ⚠️
**现象**: 评论提交成功后，无法看到帖子的历史评论记录  
**根因**: 需要检查 loadComments 函数和 API

### 问题 3: 转发按钮异常 ⚠️
**现象**: 点击转发按钮后，未弹出输入转发链接的界面  
**根因**: 分享弹窗可能被其他页面遮挡（z-index 问题）

### 问题 4: 返回按钮位置 ⚠️
**现象**: 返回按钮在顶部  
**要求**: 返回按钮应放置在最下方，所有交互按钮统一放置在最下方  
**原因**: 用户从上面往下看，看到最下边时才要发生交互行为

## 修复方案

### 问题 1 修复：观看量逻辑
**当前逻辑**:
```javascript
// 每次 loadPost() 都会调用 GET /api/posts/:id
// API 中每次都增加 view_count
```

**修复方案**:
1. 只在首次打开帖子时增加观看量
2. 点赞/评论后重新加载帖子时不增加观看量
3. 使用 sessionStorage 记录是否已观看

**实现**:
```javascript
// 前端：检查是否已观看
const hasViewed = sessionStorage.getItem(`viewed_${postId}`);
if (!hasViewed) {
  // 首次观看，API 会增加 view_count
  sessionStorage.setItem(`viewed_${postId}`, 'true');
}

// 或者：API 层面区分首次观看和重新加载
GET /api/posts/:id?viewed=true  // 重新加载，不增加观看量
```

### 问题 2 修复：评论显示

2. API 返回的评论数据格式
3. 评论列表渲染逻辑

### 问题 3 修复：分享弹窗 z-index
**当前**: share-modal z-index: 2000  
**检查**: 是否有其他元素 z-index > 2000

### 问题 4 修复：返回按钮位置

**修复**: 移动返回按钮到底部操作栏
