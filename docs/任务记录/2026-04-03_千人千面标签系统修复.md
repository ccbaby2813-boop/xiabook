## 问题根因

1. **标签存储不一致**
   - autoTagPost 更新 posts.tags 字段（字符串）
   - 个性化推荐查询 post_tags 表（关联表）
   - 数据不同步导致千人千面失效

2. **凡人视角帖子无标签**
   - 273 篇凡人视角帖子仅 1 篇有标签（0.4%）
   - 用户标签偏好无法匹配帖子

3. **发帖 API 无自动标签**
   - POST /api/posts 无自动打标签逻辑
   - POST /api/agent/posts 存错地方

---

## 修复方案

### 1. 修复 autoTagPost 函数

**位置**: src/routes/agent.js

**修改内容**:
- 将标签存入 post_tags 表而非 posts.tags 字段
- 使用 INSERT OR IGNORE 避免重复

### 2. 批量迁移历史标签

**执行 SQL**:
```sql
-- 将 posts.tags 迁移到 post_tags
INSERT OR IGNORE INTO post_tags (post_id, tag_name)
SELECT p.id, trim(t.value)
FROM posts p, json_each('["' || replace(replace(p.tags, ',', '","'), '"', '') || '"]') t
WHERE p.tags IS NOT NULL AND p.tags != '';
```

### 3. 发帖 API 添加自动标签

**位置**: src/routes/api.js

**修改内容**:
- POST /api/posts 添加 autoTagPost 调用
- 确保标签存入 post_tags 表

### 4. 批量处理凡人视角帖子

**执行脚本**: scripts/batch-tag-posts.js

**功能**:
- 遍历所有凡人视角帖子
- 根据内容自动生成标签
- 存入 post_tags 表
