## 📋 任务背景

**问题**: 新用户注册成功后，页面显示"加载中..."但帖子不加载，控制台报错。

**根因**: 页面重构后删除了某些功能模块（侧边栏、圈子、消息等），但 JS 代码仍引用这些不存在的 DOM 元素，导致 `getElementById` 返回 `null`，后续调用 `classList` 等方法时报错，JS 执行中断。

---

## 🔍 排查过程

### 第一阶段：零散修复（19:00-19:30）

| 时间 | 错误 | 修复 | 版本 |
|------|------|------|------|
| 18:26 | `closeModal is not defined` | 改为 `closeModalWithLayer` | v12 |
| 18:29 | `Cannot read properties of null (reading 'classList')` | 添加元素检查 | v13 |
| 18:48 | 同上 | 修复 `initSubTabs` | v14 |
| 18:53 | 同上 | 修复 `initLogoDropdown` | v15 |

**问题**: 治标不治本，修复一个错误又出现新的。

### 第二阶段：深度排查（19:30-20:00）

**发现**: 错误都指向同一个根因 - **JS 引用了不存在的 DOM 元素**。

**排查命令**:
```bash
# 提取 HTML 中所有 ID
curl -s "https://xiabook.cn/" | grep -o 'id="[^"]*"' | sort > html_ids.txt

# 提取 JS 中所有 getElementById
grep -oP "getElementById\('\K[^']+(?=')" public/js/app.js | sort > js_ids.txt

# 找出差异
comm -23 js_ids.txt html_ids.txt
```

**结果**: 发现 13 个 JS 引用但 HTML 不存在的 ID。

### 第三阶段：系统性修复（20:00-20:20）

**修复策略**:
1. 已移除功能 → 添加 `return` 提前返回
2. 已移除元素 → 设为 `null` 并添加注释
3. ID 改名 → 更新为正确的 ID
4. 动态创建 → 无需修复（运行时存在）

## 🎓 经验总结

### 教训
1. **页面重构后必须同步清理 JS 代码**
2. **所有 DOM 操作前必须检查元素是否存在**
3. **不要零散修复，要系统性排查**

### 最佳实践
1. **自动化检查脚本** - 每次重构后执行
2. **防御性编程** - 所有 `getElementById` 后加检查
3. **版本号管理** - 使用时间戳强制刷新
4. **文档同步** - 修复后立 即更新文档
