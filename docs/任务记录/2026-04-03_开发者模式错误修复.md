## 错误列表（开发者模式截图）

### 错误 1: 默认头像 404 ⚠️
**现象**: GET /images/default-avatar.png 404 Not Found  
**影响**: 用户头像显示为空白  
**优先级**: P2

### 错误 2: API 网关超时 ⚠️
**现象**: GET /api/posts?category=AI视角... 504 Gateway Time-out  
**影响**: 带 category 参数的 API 请求超时  
**优先级**: P1

### 错误 3: API 返回 HTML 而非 JSON ⚠️
**现象**: [loadPosts] SyntaxError: Unexpected token '<'  
**根因**: 504 错误页面是 HTML，前端期望 JSON  
**优先级**: P1

### 错误 4: CSS @import 位置错误 ⚠️
**现象**: Define @import rules at the top of the stylesheet  
**影响**: CSS 警告，不影响功能  
**优先级**: P3

### 错误 5: 表单字段缺少 label ⚠️
**现象**: No label associated with a form field  
**影响**: 可访问性警告  
**优先级**: P3

## 修复方案

### 错误 1 修复：创建默认头像图片 ✅
**方案**: 创建 default-avatar.svg，修复头像渲染逻辑

**根因**:
- 数据库中很多用户 avatar 字段存的是 emoji（🦞🌈等）
- 前端代码直接把 avatar 当作图片 URL 加载
- 导致浏览器尝试请求 emoji 的 URL 编码（%F0%9F...）

**修复**:
1. 创建 default-avatar.svg 默认头像
2. 更新 app.js 中 default-avatar.png 路径为.svg
3. 修复 4 处头像渲染逻辑，添加长度检查
   - avatar.length > 2 才当作图片加载
   - emoji（长度≤2）直接显示

### 错误 2 修复：API 网关超时
**方案**: 
1. 检查 nginx 配置
2. 检查 API 响应时间
3. 增加超时时间或优化查询

### 错误 3 修复：API 错误处理
**方案**:
1. 前端添加错误处理
2. 检查返回内容类型
3. 显示友好错误提示

### 错误 4 修复：CSS @import 位置
**方案**: 移动 @import 到 CSS 文件顶部

### 错误 5 修复：表单 label
**方案**: 添加 aria-label 或关联 label
