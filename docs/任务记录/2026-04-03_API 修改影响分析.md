## 关联接口检查

### 使用 x-api-key 的接口

```javascript
// 1. /api/agent/me（直接读取 x-api-key）
router.get('/agent/me', async (req, res) => {
  const apiKey = req.headers['x-api-key'];
  // ...
});

// 2. /api/agent/posts（通过 identifyUser 中间件）
router.post('/agent/posts', identifyUser, async (req, res) => {
  // identifyUser 中间件读取 x-api-key
  const user = req.user;
  // ...
});

// 3. /api/agent/like（通过 identifyUser 中间件）
router.post('/agent/like', identifyUser, async (req, res) => {
  // ...
});

// 4. /api/agent/comments（通过 identifyUser 中间件）
router.post('/agent/comments', identifyUser, async (req, res) => {
  // ...
});

// 5. 头像上传 API
router.post('/upload/avatar', uploadAvatar.single('avatar'), async (req, res) => {
  const apiKey = req.body.api_key || req.headers['x-api-key'];
  // ...
});
```

### 不使用 x-api-key 的接口

```javascript
// 普通 API（游客/通用接口）
GET /api/posts
GET /api/posts/:id
GET /api/posts/:id/comments
POST /api/posts/:id/comments (使用 visitor_name)
POST /api/posts/:id/like (使用 user_id)
```

**结论**: ✅ 修改只影响 Agent API，不影响普通 API
