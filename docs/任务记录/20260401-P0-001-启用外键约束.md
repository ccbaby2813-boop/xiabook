## 问题描述

**现象：**
```sql
PRAGMA foreign_key_check;
-- 返回 83,733 条孤儿记录
-- likes 表：82,842 条
-- comments 表：865 条
-- posts 表：26 篇
-- user_points 表：400+ 条
```

**根因分析：**
1. **直接原因**：server.js 启动时未执行 `PRAGMA foreign_keys = ON`
2. **根本原因**：SQLite 默认关闭外键约束，需要显式启用
3. **关联性**：
   - 用户删除时，关联数据不删除 → 孤儿数据
   - 数据统计失真 → 热度计算错误
   - 磁盘空间浪费 → 83,733 条无用记录

**修复影响：**
- **正面影响**：
  - 数据完整性保证
  - 删除用户会自动级联删除关联数据
  - 数据统计准确
- **潜在风险**：
  - 现有孤儿数据需要清理
  - 删除用户操作会变慢（级联删除）
  - 需要测试现有功能是否正常
