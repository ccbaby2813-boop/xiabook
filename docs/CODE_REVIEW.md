## 📋 审查流程

### 1. 提交前自检

开发者在提交代码前需完成：

- [ ] 代码符合 CODE_STYLE.md 规范
- [ ] 添加必要的注释
- [ ] 运行测试用例
- [ ] 更新相关文档
- [ ] 无 console.log 调试代码

### 2. 创建 Pull Request

PR 模板：

```markdown
## 变更说明
- 修复了 XXX 问题
- 添加了 XXX 功能

## 测试情况
- [ ] 单元测试通过
- [ ] 手动测试通过

## 影响范围
- 影响的模块：XXX
- 是否需要数据库迁移：是/否
```

### 3. 代码审查

审查人检查项：

- [ ] 代码逻辑正确
- [ ] 符合代码规范
- [ ] 无安全漏洞
- [ ] 性能影响可接受
- [ ] 测试覆盖率达标

### 4. 合并代码

审查通过后：

- [ ] 解决所有评论
- [ ] 重新运行测试
- [ ] 合并到 master 分支
- [ ] 删除功能分支

---

## 🔍 审查清单

### 安全性

- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 无 XSS 风险（使用 xss 过滤）
- [ ] 无 CSRF 风险（使用 CSRF Token）
- [ ] 敏感信息不硬编码
- [ ] 密码/密钥已脱敏

### 性能

- [ ] 无 N+1 查询问题
- [ ] 大数据量使用分页
- [ ] 频繁查询使用缓存
- [ ] 无内存泄漏风险

### 可维护性

- [ ] 函数长度<50 行
- [ ] 文件长度<500 行
- [ ] 有必要的注释
- [ ] 命名清晰易懂
