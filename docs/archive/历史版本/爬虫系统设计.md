## 二、防重复机制

### 2.1 进度记录文件

每个爬虫都有独立的进度记录文件：

**海外洋虾爬虫**:
- 文件: `data/crawler_moltbook_progress.json`
- 内容:
```json
{
  "lastCrawlTime": "2026-03-16T12:00:00.000Z",
  "lastPostId": "moltbook_12345",
  "crawlCount": 150
}
```

**凡人视角爬虫**:
- 文件: `data/crawler_human_progress.json`
- 内容:
```json
{
  "lastCrawlTime": "2026-03-16T17:00:00.000Z",
  "sources": {
    "tieba": { "lastCrawlTime": "...", "count": 5 },
    "hupu": { "lastCrawlTime": "...", "count": 3 }
  },
  "totalCrawlCount": 500,
  "dailyCrawlCount": 8,
  "lastCrawlDate": "2026-03-16"
}
```

### 2.2 去重策略

1. **URL去重**: 记录已爬取的original_url，重复URL跳过
2. **内容哈希**: 对内容计算MD5，重复内容跳过
3. **时间窗口**: 只爬取最近24小时的内容
4. **每日限制**: 凡人视角爬虫每天最多爬取50条

### 2.3 数据库去重

```sql
-- 插入前检查是否存在
INSERT INTO moltbook_posts (title, content, author, original_url)
SELECT ?, ?, ?, ?
WHERE NOT EXISTS (
  SELECT 1 FROM moltbook_posts WHERE original_url = ?
);
```

## 五、内容处理流程

### 5.1 海外洋虾爬虫流程

```
1. 读取进度文件
   ↓
2. 调用Moltbook API/爬取网页
   ↓
3. 提取帖子内容（标题、正文、作者、URL）
   ↓
4. 翻译内容（英文→中文）
   ↓
5. 去重检查（URL+内容哈希）
   ↓
6. 存入moltbook_posts表
   ↓
7. 更新进度文件
   ↓
8. 记录日志
```

### 5.2 凡人视角爬虫流程

```
1. 读取进度文件
   ↓
2. 检查今日是否已爬取
   ↓
3. 遍历内容源（贴吧→虎扑→小红书）
   ↓
4. 爬取各平台内容
   ↓
5. 内容清洗（去广告、格式化）
   ↓
6. 去重检查
   ↓
7. 存入posts表（category='凡人视角'）
   ↓
8. 更新进度文件
   ↓
9. 记录日志
```
