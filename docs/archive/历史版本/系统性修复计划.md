## 📋 修复原则（老板要求）

### 1. 根因分析（强制）

每个问题修复前必须回答：
- **根因是什么？** - 为什么会出现这个问题
- **关联性是什么？** - 与哪些模块/数据/功能相关
- **修复影响是什么？** - 修复后会影响什么，有什么风险

### 2. 遵循 SOP 流程（强制）

严格按照 `xiabook-sop/SKILL.md` 执行：
1. 查看 CHECKLIST.md
2. 查阅 INDEX.md 定位章节
3. **Step 2.5 关联性分析**（新增，强制）
4. 查阅相关章节并输出结果
5. 回答三个问题并引用查阅结果
6. 填写任务模板
7. 执行开发任务
8. 五宝验收
9. 更新文档 + Git 提交

### 3. 文档更新（强制）

每次修复后必须更新：
- `docs/CHECKLIST.md` - 改动记录
- `docs/虾书开发手册_v1.0_*.md` - 相关章节
- `docs/任务记录/YYYYMMDD-任务名.md` - 任务详情
- `HEARTBEAT.md` - 当前状态

### 4. 连续性保证（强制）

**问题：** 超出 session 上下文后如何保证连续性？

**解决方案：**

#### 4.1 详细记录机制

**不写摘要，写完整细节！**

```markdown
# 任务记录：20260401-启用外键约束

## 问题描述
- 现象：PRAGMA foreign_key_check 返回 83,733 条孤儿记录
- 根因：server.js 启动时未执行 PRAGMA foreign_keys = ON
- 关联影响：
  - likes 表 82,842 条孤儿记录（用户删除未级联）
  - comments 表 865 条孤儿记录
  - posts 表 26 篇孤儿帖子
  - user_points 表 400+ 条孤儿记录

## 修复方案
1. server.js 启动时添加 db.run("PRAGMA foreign_keys = ON")
2. 清理现有孤儿数据
3. 验证外键约束生效

## 修复步骤
1. 备份数据库：cp data/xiabook.db data/xiabook_backup_20260401.db
2. 修改 server.js 第 X 行，添加...
3. 执行清理 SQL：DELETE FROM likes WHERE...
4. 重启服务器
5. 验证：PRAGMA foreign_key_check 应返回空

## 验证结果
- [ ] PRAGMA foreign_keys = 1
- [ ] PRAGMA foreign_key_check 返回空
- [ ] 新功能正常（删除用户会级联删除）

## 文档更新
- [ ] CHECKLIST.md 已更新
- [ ] 开发手册第七章已更新
- [ ] 任务记录已创建
```
