## 二、敏感词库

### 2.1 词库位置
- 文件: `/config/sensitive_words.json`
- 格式: JSON，按分类组织

### 2.2 分类说明

| 分类 | 拦截策略 | 说明 |
|------|----------|------|
| illegal | **强制拦截** | 毒品、赌博、诈骗、传销、黑客等违法行为 |
| porn_violence | **强制拦截** | 色情、强奸、暴力虐待等内容 |
| attack | **建议拦截** | 人身攻击、辱骂词汇 |
| spam | **建议拦截** | 广告spam、虚假兼职 |
| political | **极端情况拦截** | 仅保留分裂、暴恐等极端内容 |

### 2.3 词库原则

**已删除的模糊词汇**:
- ❌ 领导人、主席、总书记、政府、民主、自由
- ❌ 抗议、示威、罢工、集会
- ❌ 翻墙、VPN、境外

**保留的底线词汇**:
- ✅ 台独、藏独、疆独、香港独立
- ✅ 暴恐、恐怖袭击
- ✅ 颠覆国家政权

## 三、自动审核实现

### 3.1 审核代码示例

```javascript
// 加载敏感词库
const sensitiveWords = require('./config/sensitive_words.json');

// 审核函数
function checkContent(content) {
  const result = {
    passed: true,
    level: 'pass',      // pass / warning / block
    reasons: [],
    matchedWords: []
  };
  
  // 1. 强制拦截类
  const blockWords = [
    ...sensitiveWords.illegal.words,
    ...sensitiveWords.porn_violence.words,
    ...sensitiveWords.political.words
  ];
  
  for (const word of blockWords) {
    if (content.includes(word)) {
      result.passed = false;
      result.level = 'block';
      result.reasons.push('包含违规内容');
      result.matchedWords.push(word);
      return result;
    }
  }
  
  // 2. 警告类
  const warningWords = [
    ...sensitiveWords.attack.words,
    ...sensitiveWords.spam.words
  ];
  
  for (const word of warningWords) {
    if (content.includes(word)) {
      result.level = 'warning';
      result.reasons.push('包含不当内容');
      result.matchedWords.push(word);
      // 不拦截，但记录
    }
  }
  
  return result;
}
```

### 3.2 API集成

```javascript
// POST /api/posts 创建帖子
app.post('/api/posts', async (req, res) => {
  const { content } = req.body;
  
  // 自动审核
  const checkResult = checkContent(content);
  
  if (checkResult.level === 'block') {
    return res.status(400).json({
      error: '内容包含违规信息，无法发布',
      details: checkResult.reasons
    });
  }
  
  // 警告类允许发布，但标记待人工复核
  const needsReview = checkResult.level === 'warning';
  
  // 保存帖子
  const post = await db.createPost({
    ...req.body,
    status: needsReview ? 'pending_review' : 'active',
    audit_log: JSON.stringify(checkResult)
  });
  
  res.json({ success: true, post, needsReview });
});
```

## 四、人工审核

### 4.1 审核后台

**位置**: admin.html → "内容审核" 模块

**功能**:
- 待审核内容列表
- 一键通过 / 拒绝
- 批量操作
- 审核记录查询

### 4.2 审核流程

```
内容进入待审核队列
       ↓
审核员查看内容
       ↓
┌─────────────┐
│ 通过 → 发布上线 │
│ 拒绝 → 返回用户 │
│ 删除 → 记录违规 │
└─────────────┘
```

### 4.3 审核员权限

| 角色 | 权限 |
|------|------|
| 超级管理员 | 全部审核权限 |
| 内容审核员 | 审核帖子、评论 |
| 只读观察员 | 查看审核记录 |

## 八、审核原则总结

### 8.1 必须拦截
- 毒品、赌博、诈骗等违法行为
- 色情、暴力内容
- 分裂国家、恐怖主义

### 8.2 建议拦截
- 人身攻击、辱骂
- 广告spam

### 8.3 正常放行
- 政治讨论（非极端）
- 社会话题
- 技术交流
- 生活分享

### 8.4 核心原则
> **"违法违规必拦，正常讨论放行"**
