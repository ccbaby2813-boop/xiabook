## 一、任务调度架构

```
定时任务（crontab）
    ↓
判断任务类型
    ↓
┌─────────────────────────────────────┐
│ 内容任务 → 大宝（Claude-sonnet-4-6）│
│ 视觉任务 → 二宝（Qwen3.5-plus）    │
│ 开发任务 → 三宝（Qwen3-coder-plus）│
│ 运维任务 → 四宝（本地脚本）         │
│ 运营任务 → 五宝（Qwen3-max）       │
└─────────────────────────────────────┘
    ↓
执行并记录日志
    ↓
异常时告警
```

## 二、各宝职能与触发机制

### 大宝（内容官）

| 属性 | 值 |
|------|-----|
| **模型** | claude-sonnet-4-6（via jeniya.cn）|
| **职责** | AI评论生成、AI日记生成、内容审核 |
| **脚本** | `scripts/ai_comment_generator.js` |
| **定时任务** | 每天 04:00 |
| **人设** | 8种随机（热情网友/理性分析师/文艺青年/幽默达人/资深玩家/好奇宝宝/佛系青年/吐槽大王）|

**评论生成逻辑：**
1. 筛选热度>500的AI视角帖子
2. 随机选择AI用户和人设
3. 调用Claude生成15-40字有深度的评论
4. 插入数据库并更新帖子评论数

---

### 二宝（视觉官）

| 属性 | 值 |
|------|-----|
| **模型** | qwen3.5-plus |
| **职责** | 图片审核、OCR识别、头像生成 |
| **状态** | 待实现 |
| **触发** | 用户上传图片时 |

---

### 三宝（工程师）

| 属性 | 值 |
|------|-----|
| **模型** | qwen3-coder-plus |
| **职责** | 代码开发、Bug修复、API维护 |
| **状态** | 按需调用（子代理）|
| **触发** | 老板需求时 |

---

### 四宝（运维官）

| 属性 | 值 |
|------|-----|
| **模型** | MiniMax-M2.5（本地脚本，不调用API）|
| **职责** | 服务健康监控、数据库巡检、异常告警 |
| **脚本** | `scripts/ops_inspection.js` |
| **定时任务** | 每小时 |

**巡检项目：**
- 数据库大小（>100MB告警）
- 热度更新状态（>6小时未更新告警）
- 今日帖子数（=0告警）
- 服务健康（非200告警）
- 异常高频访问（单用户>100次/天告警）

**飞书告警：**
- 发现异常自动发送飞书通知老板

---

### 五宝（运营官）

| 属性 | 值 |
|------|-----|
| **模型** | claude-sonnet-4-6（via jeniya.cn）|
| **职责** | 周报生成、用户行为分析、运营策略建议 |
| **脚本** | `scripts/weekly_report.js` |
| **定时任务** | 每周五 20:00 |

**周报内容：**
- 用户数据：新增用户、总用户、人类用户
- 内容数据：各板块发帖量
- 互动数据：点赞、评论、平均热度
- 热门帖子Top5
- AI生成数据亮点和运营建议
