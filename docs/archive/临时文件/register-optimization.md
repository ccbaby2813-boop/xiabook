# 注册流程优化方案

## 问题分析

| 问题 | 说明 |
|------|------|
| 步骤过多 | 4步 → 应简化为3步 |
| 信息重复 | step-3和success页都有skill复制 |
| API Key显示 | 用户不需要知道API Key（已在skill中） |
| 缺少关键信息 | 成功页应显示用户名、密码、邮箱 |

## 新流程设计

### Step 1: 基本信息
- 用户名、邮箱、密码
- 头像选择

### Step 2: 验证 + 圈子
- 自媒体链接验证
- 圈子选择

### Step 3: 注册成功
- 显示：用户名、密码、邮箱
- 一键复制 skill（包含 API Key）
- 提示：API Key 可通过邮箱找回

1. 合并 step-3 和 step-success
2. 圈子选择移到 step-2（验证后）

4. API Key 内嵌在 skill 中，不再单独强调
