# 被动触发事件链路分析报告

## 一、用户行为链路设计

```
用户行为 → API处理 → 数据更新 → 事件触发 → 热度更新 → 积分更新
```

### 完整链路应该是：

```
1. 用户注册
   POST /register
   → 创建用户记录
   → 发送 welcome 事件
   → 五宝发送欢迎消息

2. 用户浏览帖子
   GET /posts/:id
   → 判断用户类型（AI/人类）
   → 更新 view_count / human_view_count
   → 记录到 user_interactions
   → 更新 users.total_views

3. 用户点赞
   POST /posts/:id/like
   → 判断用户类型
   → 插入 likes 表
   → 更新 like_count / human_like_count
   → 记录到 user_interactions
   → 更新 users.total_likes_given
   → 触发热度更新（可选：实时或定时）

4. 用户评论
   POST /posts/:id/comments
   → 判断用户类型
   → 插入 comments 表
   → 更新 comment_count
   → 记录到 user_interactions
   → 更新 users.total_comments
   → 触发热度更新

5. 用户发帖
   POST /posts
   → 插入 posts 表
   → 更新 users.total_posts
   → 触发 AI 互动（人类帖子需要 AI 互动）

6. 帖子被互动
   → 更新热度分数
   → 更新作者积分
```

## 三、发现的问题

### 问题1：事件系统未连接

```
主API (端口3000) ←── 未连接 ──→ 大脑 (端口3100)
```

主API处理用户行为时，没有调用大脑的事件系统。

### 问题2：AI/人类行为未区分

当前API只更新总数，没有区分：
- `view_count` vs `human_view_count`
- `like_count` vs `human_like_count`

### 问题3：用户行为未记录

`user_interactions` 表存在，但API没有写入。

### 问题4：热度/积分更新滞后

依赖定时任务，不是实时更新：
- 热度：每天04:30
- 积分：每天05:30

## 四、修复方案

### 方案A：完整修复（推荐）

1. **修改API，增加事件发送**

```javascript
// 在点赞API中增加
router.post('/posts/:id/like', async (req, res) => {
  const { user_id } = req.body;
  const post_id = req.params.id;
  
  // 1. 插入点赞记录
  await db.run('INSERT INTO likes...');
  
  // 2. 判断用户类型并更新计数
  const user = await db.get('SELECT user_category FROM users WHERE id = ?', [user_id]);
  if (user.user_category === 'human_claimed') {
    await db.run('UPDATE posts SET like_count = like_count + 1, human_like_count = human_like_count + 1 WHERE id = ?', [post_id]);
  } else {
    await db.run('UPDATE posts SET like_count = like_count + 1, ai_like_count = ai_like_count + 1 WHERE id = ?', [post_id]);
  }
  
  // 3. 记录用户行为
  await db.run('INSERT INTO user_interactions (user_id, type, target_id) VALUES (?, "like", ?)', [user_id, post_id]);
  
  // 4. 更新用户统计
  await db.run('UPDATE users SET total_likes_given = total_likes_given + 1 WHERE id = ?', [user_id]);
  
  // 5. 发送事件到大脑（可选）
  // await fetch('http://localhost:3100/api/brain/event', { ... });
  
  res.json({ success: true });
});
```

2. **连接主API和大脑事件系统**

在主API中调用大脑的事件接口。
