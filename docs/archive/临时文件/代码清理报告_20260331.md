## 🎯 后续建议

### 1. 建立代码审查清单（✅ 已实施）
每次页面重构后，执行以下命令检查残留：
```bash
# 提取 HTML 中所有 ID
curl -s "https://xiabook.cn/" | grep -o 'id="[^"]*"' | sed 's/id="//;s/"//' | sort > html_ids.txt

# 提取 JS 中所有 getElementById
grep -oP "getElementById\('\K[^']+(?=')" public/js/app.js | sort | uniq > js_ids.txt

# 找出差异
comm -23 js_ids.txt html_ids.txt
```

### 2. 添加防御性编程规范（✅ 已实施）
```javascript
// 所有 DOM 操作前必须检查元素
const element = document.getElementById('xxx');
if (!element) {
  console.log('[xxx] 元素不存在，跳过');
  return;
}
```

### 3. 定期清理（✅ 已实施）
建议每月执行一次代码清理，删除：
- 已移除功能的 JS 代码
- 未使用的 CSS 样式
- 过时的 API 调用

### 4. 版本号管理（✅ 已实施）
```bash
# 每次修改后更新时间戳强制刷新
TIMESTAMP=$(date +%s)
sed -i "s/app.js?v=[^\"]*/app.js?v=$TIMESTAMP/" public/index.html
sed -i "s/style.css?v=[^\"]*/style.css?v=$TIMESTAMP/" public/index.html
```
