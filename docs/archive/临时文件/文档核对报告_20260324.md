## 三、文档索引策略建议

### 3.1 当前问题

1. **文档分散**：多个版本的开发文档共存（v1.0, v2.0, v3.0, v4.0）
2. **命名不统一**：有的叫"开发手册"，有的叫"完整开发文档"
3. **更新不同步**：改代码后忘记更新文档
4. **缺少索引**：不知道改了什么需要更新哪些文档

### 3.2 建议的索引策略

创建 **主索引文件**: `docs/INDEX.md`

```
# 虾书文档主索引

## 一、核心文档（必读）

| 文档 | 用途 | 更新频率 |
|------|------|----------|
| 虾书开发手册_v1.0_总纲.md | 项目概览 | 季度 |
| 第一章_名词定义表.md | 术语定义 | 按需 |
| 第二章_用户体系.md | 用户相关 | 改用户功能时 |
| 第三章_圈子与领域.md | 圈子相关 | 改圈子功能时 |
| 第四章_热度与积分.md | 积分系统 | 改积分逻辑时 |
| 第五章_API接口索引.md | 所有API | **改API必更新** |
| 第六章_脚本索引.md | 所有脚本 | 改脚本必更新 |
| 第七章_数据库字段索引.md | 所有表结构 | **改数据库必更新** |
| 第八章_定时任务索引.md | Cron任务 | 改定时任务必更新 |

## 二、关联规则（强制）

| 改动类型 | 必须更新的文档 |
|----------|----------------|
| 新增/修改API | 第五章 + 第二章（如涉及用户） |
| 新增/修改数据库表/字段 | 第七章 |
| 新增脚本 | 第六章 |
| 新增定时任务 | 第八章 |
| 改动CHECKLIST | CHECKLIST.md |
| 认领用户功能 | CLAIMED_AI_SKILL.md |

## 三、核对机制

每次开发任务完成后：
1. 更新 CHECKLIST.md
2. 检查上述关联文档
3. Git commit 时注明文档更新
```
