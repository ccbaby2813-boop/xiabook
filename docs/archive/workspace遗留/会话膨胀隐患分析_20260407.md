**分析时间**: 2026-04-07 22:45

## 🔴 隐患 1：Compaction 不主动触发

**现象**: 飞书会话 257KB，但 compaction 记录为 0

**根因**:
- Token 估算：54,650（43% 使用率）
- Compaction 触发阈值：70-80%
- 当前远低于阈值，所以 **compaction 不运行**

**隐患**:
- 会话持续增长，每条消息平均 1.5KB
- 预测 500 条后达 731KB
- Compaction 配置（keepRecentTokens: 10000）**只在触发后生效**

**解决方案**:
- 方案 A：降低 compaction 触发阈值（需配置 triggerTokens）
- 方案 B：增加主动截断策略（定期清理脚本）
- 方案 C：会话上限监控（超过 100KB 告警）

## 🔴 隐患 2：定时任务 delivery 配置错误

## 🔴 隐患 5：工具调用返回大量数据

**案例**: 飞书会话最大 toolResult 123KB（exec 检查 API 响应）

**根因**: 违反 MEMORY.md 规则 8（exec 输出 >5KB 只保留摘要）

**隐患**: 每次 exec 返回大数据，存入会话后无法自动压缩

## 📊 综合解决方案

### 第一层：Compaction 配置优化 ✅
```json
{
  "mode": "safeguard",
  "reserveTokens": 50000,
  "keepRecentTokens": 10000,
  "maxHistoryShare": 0.3
}
```
**已生效，但需要触发阈值才会运行**

### 第二层：定时任务 delivery 修复
```bash
# 所有后台自动化任务
openclaw cron update <job-id> --delivery '{"mode": "none"}'
```

### 第三层：会话监控脚本
- 每 4 小时检查飞书会话大小
- 超过 100KB 告警
- 超过 500KB 自动截断

### 第四层：MEMORY.md 规则强化 ✅
- exec 输出 >5KB 只保留摘要
- 工具结果先写文件再读摘要
