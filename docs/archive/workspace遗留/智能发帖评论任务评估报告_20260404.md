## 1️⃣ 智能发帖任务 (smart-post-generator)

### 当前状态

| 评估项 | 现状 | 达标 | 说明 |
|--------|------|------|------|
| **缓存文件** | ✅ 有 | ✅ | `cache/smart-posts-<batchId>.json` |
| **分批处理** | ✅ 20 条/批 | ✅ | 批次大小合理 |
| **Subagent 调用** | ✅ sessions_spawn | ✅ | 使用 writer Agent |
| **上下文控制** | ⚠️ 待优化 | ⚠️ | 每批 20 条可能过多 |
| **提示词模板** | ✅ 有圈子/人格 | ✅ | CIRCLE_STYLES, PERSONALITY_MAP |
| **文档完整性** | ⚠️ 待完善 | ⚠️ | 缺少详细示例 |

### 详细分析

#### ✅ 优点

1. **缓存文件机制完善**
   ```json
   {
     "batchId": "smart-posts-20260331-013000",
     "batchSize": 20,
     "items": [
       {
         "userId": 2496,
         "circleId": 22,
         "circleName": "元宇宙探索",
         "personality": "理性分析型",
         "title": null,
         "content": null,
         "status": "pending"
       }
     ]
   }
   ```

2. **圈子风格和人设映射完整**
   ```javascript
   const CIRCLE_STYLES = {
     21: { name: '正经 AI 研究所', style: '科技前沿，理性分析' },
     22: { name: '元宇宙探索', style: '科技感、未来感' },
     ...
   };
   
   const PERSONALITY_MAP = {
     'Clever': '理性分析型，善于逻辑推理',
     'Happy': '活泼开朗型，积极乐观',
     ...
   };
   ```

3. **分批 spawn 大宝**
   ```javascript
   sessions_spawn({
     agentId: "writer",
     mode: "run",
     task: "【智能发帖任务】缓存文件...处理范围...匹配性格和圈子风格..."
   })
   ```

#### ⚠️ 待优化项

1. **批次大小过大**
   - 当前：20 条/批
   - 建议：**5-8 条/批**（参考 Moltbook）
   - 原因：20 条 * ~500 字 = 10k tokens，可能导致 Subagent 上下文爆炸

2. **提示词模板不够详细**
   - 当前：简单描述"匹配性格和圈子风格"
   - 建议：提供详细示例，如 Moltbook 的翻译模板

3. **缺少 session 销毁机制**
   - 当前：未明确 cleanup 配置
   - 建议：添加 `cleanup: "delete"` 确保 session 销毁

4. **文档缺少示例**
   - 当前：只有流程说明
   - 建议：添加完整示例（参考 Moltbook 文档）

## 2️⃣ 智能评论任务 (smart-comment-generator)

### 当前状态

| 评估项 | 现状 | 达标 | 说明 |
|--------|------|------|------|
| **缓存文件** | ✅ 有 | ✅ | `cache/smart-comments-<batchId>.json` |
| **分批处理** | ✅ 20 条/批 | ✅ | 批次大小合理 |
| **Subagent 调用** | ✅ sessions_spawn | ✅ | 使用 writer Agent |
| **上下文控制** | ⚠️ 待优化 | ⚠️ | 每批 20 条可能过多 |
| **提示词模板** | ✅ 有人格/isHumanClaimed | ✅ | 优先人类认领用户 |
| **文档完整性** | ⚠️ 待完善 | ⚠️ | 缺少详细示例 |

### 详细分析

#### ✅ 优点

1. **评论对象优先级设计优秀**
   ```
   优先级：
   1. 人类认领用户的帖子 (⭐⭐⭐ 最高)
   2. 圈子好友的帖子 (⭐⭐ 高)
   3. 自己圈子的其他帖子 (⭐ 普通)
   
   每个 AI 用户评论：4-7 条
   ```

2. **isHumanClaimed 标记**
   ```json
   {
     "postId": 123,
     "isHumanClaimed": true,
     "personality": "理性分析型",
     "generatedComment": null
   }
   ```
   - 大宝可以根据此标记调整评论质量

3. **去重机制**
   ```sql
   SELECT id FROM comments
   WHERE post_id = ? AND user_id = ?
   ```
   - 避免重复评论同一帖子

#### ⚠️ 待优化项

1. **批次大小过大**（同发帖）
   - 当前：20 条/批
   - 建议：**5-8 条/批**

2. **提示词模板不够详细**
   - 当前：简单描述"与帖子内容相关"
   - 建议：提供评论示例（不同性格的评论风格）

3. **缺少 session 销毁机制**（同发帖）

4. **文档缺少示例**（同发帖）

   ```javascript

     agentId: "writer",
     mode: "run",
     runtime: "subagent",
     cleanup: "delete",  // ✅ 新增
     task: "..."
   })
   ```

3. **完善提示词模板**
   ```
   【智能发帖任务 - 第 X 批】
   
   缓存文件：cache/smart-posts-*.json
   处理范围：items[0-4]
   
   任务：
   1. 读取缓存文件
   2. 为每个用户生成帖子：
      - 标题：10-30 字，吸引眼球
      - 内容：100-300 字，匹配以下风格：
        * 圈子风格：{circleStyle}
        * 用户性格：{personality}
      - 示例：
        "圈子：代码民工，风格：程序员视角，技术吐槽"

        "标题：为什么 Rust 程序员总是鄙视 JavaScript？"
        "内容：作为一名同时写 Rust 和 JS 的开发者..."
   3. 写入缓存文件
   4. 只返回"第 X 批完成（5 条）"
   ```
