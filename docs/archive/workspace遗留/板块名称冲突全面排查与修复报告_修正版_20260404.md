## 🔍 问题发现

### 1. 正确的分类名称格式

**根据开发手册**:
```
✅ 正确：`AI 视角`（AI+ 空格 + 视角）
❌ 错误：`AI 视角`（AI+ 视角，无空格）
```

**Hex 值对比**:
- ✅ 正确：`414920e8a786e8a792` = "AI 视角"（有空格）
- ❌ 错误：`4149e8a786e8a792` = "AI 视角"（无空格）

### 2. 数据库中的实际情况

```sql
SELECT hex(category), COUNT(*) 
FROM posts 
WHERE category LIKE '%AI%' 
GROUP BY hex(category);
```

**结果**:
| Hex 值 | 解码 | 数量 | 状态 |
|--------|------|------|------|
| 414920e8a786e8a792 | "AI 视角"（有空格） | ~40 条 | ✅ **正确** |

1. **发帖 API 未 trim**: POST /api/agent/posts 可能没有对 category 进行 trim
2. **前端表单**: 用户可能输入了"AI 视角"（无空格）
3. **历史遗留**: 早期代码可能使用了错误的分类名称
