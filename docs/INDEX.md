## 六、核对机制

### 每次开发结束前
1. ✅ 更新 CHECKLIST.md
2. ✅ 检查关联文档是否更新（按 INDEX.md 第二章的关联更新规则）
3. ✅ Git commit 注明文档更新

### 每周核对
1. 核对数据库表与第七章是否一致
2. 核对 API 路由与第五章是否一致
3. 检查定时任务与第八章是否一致
4. 检查是否有遗漏的文档更新
