# P1-001 密码强度优化方案

**分析时间**：2026-04-02 10:35  
**优先级**：P1

---

## 🔍 当前修复的问题

### 问题 1：老用户密码未处理
- 已注册的 897 个用户中，约 100-200 个使用弱密码
- 这些账户仍然存在安全风险

### 问题 2：前端无提示
- 用户不知道密码规则
- 注册时可能多次尝试才成功

### 问题 3：无密码强度指示器
- 用户不知道密码多强
- 无法引导用户设置更强密码

### 问题 4：找回/重置密码未验证
- 只验证了注册 API
- 重置密码 API 仍可设置弱密码

---

## ✅ 优化方案

### 1. 前端添加密码规则提示

```html
<!-- register.html -->
<div class="password-rules">
  <h4>密码要求：</h4>
  <ul>
    <li>✅ 至少 8 个字符</li>
    <li>✅ 包含大写字母（A-Z）</li>
    <li>✅ 包含小写字母（a-z）</li>
    <li>✅ 包含数字（0-9）</li>
  </ul>
</div>
```

### 2. 前端添加密码强度指示器

```html
<!-- register.html -->
<div class="password-strength">
  <div class="strength-bar" id="strengthBar"></div>
  <span id="strengthText">密码强度：未输入</span>
</div>

<script>
function checkPasswordStrength(password) {
  let score = 0;
  if (password.length >= 8) score++;
  if (/[A-Z]/.test(password)) score++;
  if (/[a-z]/.test(password)) score++;
  if (/[0-9]/.test(password)) score++;
  if (/[^A-Za-z0-9]/.test(password)) score++;
  
  const levels = ['太弱', '弱', '中', '强', '很强'];
  const colors = ['#ff4444', '#ff8800', '#ffcc00', '#88cc00', '#44cc00'];
  
  document.getElementById('strengthBar').style.width = (score * 20) + '%';
  document.getElementById('strengthBar').style.backgroundColor = colors[score];
  document.getElementById('strengthText').textContent = '密码强度：' + levels[score];
}
</script>
```

### 4. 老用户处理方案

**方案 A：强制修改（推荐）**
```sql
-- 标记弱密码用户
UPDATE users SET password_weak = 1 
WHERE length(password_hash) < 10; -- 示例条件

-- 下次登录时强制修改
if (user.password_weak) {
  return res.json({ 
    success: true, 
    require_password_change: true 
  });
}
```

**方案 B：渐进式提醒**
```javascript
// 登录成功后检查密码强度
if (password.length < 8) {
  res.json({
    success: true,
    warning: '您的密码强度较低，建议修改'
  });
}
```
