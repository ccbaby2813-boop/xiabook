
```html

  <div class="comments-title">💬 评论</div>
  <div id="comments-list"></div>
  <div class="comment-form">
    <input type="text" class="comment-input" id="comment-input">
    <button class="comment-submit" onclick="submitComment()">发送</button>
  </div>
</div>
```

**CSS 覆盖（关键）**：
```css
.comments-section {
  max-height: none !important;
  overflow: visible !important;
}
```
> ⚠️ style.css 中曾有 `.comments-section { max-height: 0 }` 导致评论区不可见，
> 必须用 `!important` 覆盖

**评论排序**：`.reverse()` 倒序显示（最新→最旧）

**手机端**：
- `.comment-form` 保持 `flex-direction: row`（输入框+按钮同行）
- `.comment-submit`：`padding: 14px 20px; font-size: 14px`
- 不使用 `flex-direction: column`（会被底部操作栏遮挡）

### 4. 无障碍 label 规范

所有表单 `<label>` 必须通过 `for` 属性关联 `<input>`：
```html
<label for="login-username">用户名</label>
<input type="text" id="login-username">
```

`aria-label` 必须有意义，不能用通用词：
- ❌ `aria-label="输入框"`
- ✅ `aria-label="搜索用户或帖子"`

### 防御性编程规范

**所有 DOM 操作前必须检查元素是否存在：**

```javascript
// ❌ 错误写法
const element = document.getElementById('xxx');
element.classList.add('active');

// ✅ 正确写法
const element = document.getElementById('xxx');
if (!element) {
  console.log('[xxx] 元素不存在，跳过');
  return;
}
element.classList.add('active');
```

### 自动化检查脚本

**每次页面重构后执行：**

```bash
# 1. 提取 HTML 中所有 ID
curl -s "https://xiabook.cn/" | grep -o 'id="[^"]*"' | sed 's/id="//;s/"//' | sort > html_ids.txt

# 2. 提取 JS 中所有 getElementById
grep -oP "getElementById\('\K[^']+(?=')" public/js/app.js | sort | uniq > js_ids.txt

# 3. 找出差异（JS 引用但 HTML 不存在）
comm -23 js_ids.txt html_ids.txt

# 4. 逐个检查并修复
```

### 版本号规范

**前端代码必须添加时间戳版本号强制刷新：**

```html
<script src="/js/app.js?v=TIMESTAMP"></script>
<link rel="stylesheet" href="/css/style.css?v=TIMESTAMP">
```

**生成命令：**
```bash
TIMESTAMP=$(date +%s)
sed -i "s/app.js?v=[^\"]*/app.js?v=$TIMESTAMP/" public/index.html
sed -i "s/style.css?v=[^\"]*/style.css?v=$TIMESTAMP/" public/index.html
```

| 层级 | z-index | 用途 |
|------|---------|------|
| 普通弹窗 | 1000 | 详情、用户信息 |
| 提示弹窗 | 1100 | alert、confirm |
| 引导弹窗 | 1200 | 新手引导、分享 |
| Toast | 2000 | 轻提示 |

**动态层级管理**：
```javascript
// 每次打开弹窗时，自动计算 z-index
let modalZIndex = 1000;

function openModal(modalId) {
  modalZIndex += 10; // 每次打开增加10
  const modal = document.getElementById(modalId);
  modal.style.zIndex = modalZIndex;
  modal.classList.add('active');
}

function closeModal(modalId) {
  document.getElementById(modalId).classList.remove('active');
  // 关闭后不减少 z-index，避免层级错乱
}
```

**规则**：
- 第一个弹窗 z-index: 1000
- 第二个弹窗 z-index: 1010
- 第三个弹窗 z-index: 1020
- 后开的弹窗始终在上层

## 十、🔴🔴🔴 CSS语法检查强制规范（2026-03-29 确立）

### 10.1 事件背景

**【老板原话】**：
> "这个问题你一定要记在文件里，开发文档也要记，你的记忆文件也要记。而且还要提醒三宝，包括五保做验收，都要排查这些问题"

**问题详情**：
- 一个空规则块 `.btn-post {` （行1249）后无任何内容
- 导致后续 **80+ 条CSS规则全部失效**
- 一个多余的 `}` （行1685）导致 @media 块提前关闭
- CSS解析器在语法错误处停止，后续所有样式不生效

**影响范围**：
- 原CSS规则数量：435条（错误状态）
- 修复后CSS规则数量：515条（恢复80条）
- footer-info-bar 的 `position: fixed` 失效，显示为 `static`

### 10.2 🔴 三宝（代码开发）必读

**每次写CSS/JS后必须执行以下检查**：

```bash
# 1. CSS括号匹配检查（必须执行！）
cd projects/xiabook/public/css
OPEN=$(grep -o '{' style.css | wc -l)
CLOSE=$(grep -o '}' style.css | wc -l)
if [ "$OPEN" != "$CLOSE" ]; then
  echo "❌ 错误：开括号=$OPEN, 关括号=$CLOSE"
  exit 1
fi
echo "✅ CSS括号匹配正常"

# 2. 检查空规则块
grep -B1 "{\$" style.css | grep -v "^--" | grep "{\$"
# 如果有输出，说明有空规则块

# 3. 检查@media块闭合
grep -c "@media" style.css
# 统计对应的闭合括号数量必须相等

# 4. 检查JS括号匹配
cd ../js
OPEN=$(grep -o '{' app.js | wc -l)
CLOSE=$(grep -o '}' app.js | wc -l)
if [ "$OPEN" != "$CLOSE" ]; then
  echo "❌ JS括号不匹配"
  exit 1
fi
```

**禁止事项**：
| 禁止项 | 原因 |
|--------|------|
| 选择器后直接写注释 | 会创建空规则块 |
| 在@media块内多余闭合 | 会提前关闭块 |
| 不检查括号匹配就提交 | 一个错误可以让80条规则失效 |

### 10.3 🔴 五宝（验收）必读

**验收时必须检查**：

1. **浏览器DevTools检查**
   ```javascript
   // 在浏览器控制台执行
   document.styleSheets[0].cssRules.length
   // 对比预期规则数量
   ```

2. **关键样式验证**
   ```javascript
   // 检查关键样式是否生效
   const footer = document.querySelector('.footer-info-bar');
   console.log(window.getComputedStyle(footer).position);
   // 应该是 'fixed'，不是 'static'
   ```

3. **移动端样式验证**
   - 切换到移动端视图（DevTools）
   - 检查 @media 块内样式是否生效

4. **对比规则数量**
   - 代码中定义的规则数 vs 实际生效的规则数
   - 如果差异大，说明有语法错误

### 10.4 快速诊断命令

**一键检查脚本**：
```bash
#!/bin/bash
echo "=== CSS/JS语法检查 ==="
CSS_FILE="public/css/style.css"
JS_FILE="public/js/app.js"

# CSS括号
CSS_OPEN=$(grep -o '{' $CSS_FILE | wc -l)
CSS_CLOSE=$(grep -o '}' $CSS_FILE | wc -l)
if [ "$CSS_OPEN" != "$CSS_CLOSE" ]; then
  echo "❌ CSS括号不匹配：开=$CSS_OPEN, 关=$CSS_CLOSE"
else
  echo "✅ CSS括号匹配：$CSS_OPEN"
fi

# JS括号
JS_OPEN=$(grep -o '{' $JS_FILE | wc -l)
JS_CLOSE=$(grep -o '}' $JS_FILE | wc -l)
if [ "$JS_OPEN" != "$JS_CLOSE" ]; then
  echo "❌ JS括号不匹配：开=$JS_OPEN, 关=$JS_CLOSE"
else
  echo "✅ JS括号匹配：$JS_OPEN"
fi

# 空规则块
EMPTY=$(grep -B1 "{\$" $CSS_FILE | grep "{\$" | wc -l)
if [ "$EMPTY" -gt 0 ]; then
  echo "⚠️ 发现 $EMPTY 个空规则块"
fi

echo "=== 检查完成 ==="
```

| **空规则块最危险** | 选择器后不能直接写注释或空行 |

## 十一、Header Grid布局规范（2026-03-30 确立）

### 11.1 问题背景

**flex布局在华为/微信浏览器上的问题**：
- 即使设置 `flex-shrink: 0`，按钮仍被压缩
- 搜索条和按钮在同一个flex容器，空间不足时互相挤压
- 多个 `@media` 重复定义Header样式，后者覆盖前者

### 11.2 最终方案：Grid布局

```css
.header-inner {
  display: grid;
  grid-template-columns: 66px minmax(62px, 1fr) 38px 38px 38px;
  align-items: center;
  gap: 4px;
}
```

### 11.3 核心原则

| 元素 | 宽度策略 | 说明 |
|------|----------|------|
| Logo | 固定 66px | 不变 |
| 搜索input | `flex:1` + `min-width:16px` | **弹性压缩** |
| 搜索按钮 | 固定 38px | 不压缩 |
| 客服按钮 | 固定 38px | 不压缩 |
| 登录按钮 | 固定 38px | 不压缩 |

### 11.4 4个响应式断点

```css
/* 小屏手机 (≤384px) */
.header-inner {
  grid-template-columns: 60px minmax(16px, 1fr) 36px 36px 36px;
  gap: 2px;
}

/* 中屏手机 (385-414px) */
.header-inner {
  grid-template-columns: 66px minmax(62px, 1fr) 38px 38px 38px;
  gap: 4px;
}

/* 超大屏手机 (≥415px) */
.header-inner {
  grid-template-columns: 66px minmax(140px, 1fr) 38px 38px auto;
  gap: 8px;
}
```

### 11.5 硬件加速（防华为/微信渲染问题）

```css
.header-inner,
.cs-button {
  -webkit-transform: translateZ(0);
  -webkit-backface-visibility: hidden;
}
```

### 11.6 禁止事项

| 禁止项 | 原因 |
|--------|------|
| 使用flex布局Header | 华为/微信浏览器渲染问题 |
| 多个@media重复定义Header | 后者覆盖前者，样式冲突 |
| 按钮用弹性宽度 | 会被压缩，导致重叠 |

### 11.7 开发检查清单

- [ ] Header使用Grid布局，不是flex
- [ ] 搜索input用 `flex:1` + `min-width:16px`
- [ ] 按钮固定38px宽度
- [ ] 只在一处定义Header样式
- [ ] 测试375px/384px/430px三种尺寸

## 9.5 帖子详情页交互规范（2026-04-03 更新）

### 9.5.1 观看量统计逻辑

**规则**：
- 每个 IP 首次打开帖子时增加观看量
- 点赞、评论等交互行为后重新加载帖子，不增加观看量
- 使用 sessionStorage 记录是否已观看

**实现**：
```javascript
async function loadPost(isReload = false) {
  const viewedKey = `viewed_${postId}`;
  const hasViewed = sessionStorage.getItem(viewedKey);
  
  // 首次观看才增加 view_count
  if (!hasViewed && !isReload) {
    sessionStorage.setItem(viewedKey, 'true');
  }
  
  const viewParam = isReload ? '?reload=1' : '';
  fetch(`${API}/posts/${postId}${viewParam}`);
}
```

**按钮顺序**：返回 | 点赞 | 评论 | 分享

## 🔴 2026-04-06 帖子详情页交互修复

### 问题背景

帖子详情页（post.html）底部操作栏交互不完善：
1. 评论输入框无 placeholder 提示
2. 评论按钮无 aria-label
3. 分享输入框无 aria-label
4. 分享按钮无 aria-label

### 修改文件

| 文件 | 改动 |
|------|------|
| `public/post.html` | 4 处交互属性补充 |

### 具体修改

**1. 评论输入框 placeholder**
```html
<!-- 修改前 -->
<textarea class="comment-input" id="commentInput"></textarea>

<!-- 修改后 -->
<textarea class="comment-input" id="commentInput" placeholder="写下你的想法..."></textarea>
```

**2. 评论按钮 aria-label**
```html
<!-- 修改前 -->
<button class="comment-submit-btn" id="commentSubmitBtn">评论</button>

<!-- 修改后 -->
<button class="comment-submit-btn" id="commentSubmitBtn" aria-label="提交评论">评论</button>
```

**3. 分享输入框 aria-label**
```html
<!-- 修改前 -->
<input type="text" class="share-link-input" id="shareLinkInput" readonly>

<!-- 修改后 -->
<input type="text" class="share-link-input" id="shareLinkInput" readonly aria-label="分享链接">
```

**4. 分享按钮 aria-label**
```html
<!-- 修改前 -->
<button class="copy-link-btn" id="copyLinkBtn">复制链接</button>

<!-- 修改后 -->
<button class="copy-link-btn" id="copyLinkBtn" aria-label="复制链接">复制链接</button>
```

### 修改原因

- **placeholder**：提示用户输入内容
- **aria-label**：无障碍访问支持（WCAG 2.1 AA 标准），屏幕阅读器可读
- 不影响视觉样式，纯功能性补充

### 验证方法

浏览器开发者工具 → Elements → 检查对应元素的属性

## 九、修改密码弹窗（2026-04-08 新增）
