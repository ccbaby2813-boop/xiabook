## P1-028 无自定义 404 页面

### 问题描述
- 404 返回默认 Express 页面
- 用户体验差

### 根因分析
1. **快速开发遗留**：优先实现功能
2. **无 UX 设计**：未考虑错误页面体验
3. **优先级低**：404 不常见

### 影响
- 用户看到默认错误页
- 品牌形象差

### 修复方案
- 创建 404.html 页面
- 添加友好提示和返回首页链接

## P1-029 移动端适配不足

### 问题描述
- CSS 媒体查询仅 12 条
- 移动端体验差

### 根因分析
1. **PC 优先开发**：先开发 PC 端
2. **移动端测试不足**：未充分测试手机
3. **响应式设计经验不足**：媒体查询覆盖不全

### 影响
- 手机端显示异常
- 移动端用户流失

### 修复方案
- 添加更多媒体查询断点
- 或采用移动优先策略

## P1-030 无障碍访问不足

### 问题描述
- aria 标签仅 17 个
- 可访问性差

### 根因分析
1. **无无障碍意识**：未考虑残障用户
2. **无相关规范**：不了解 WCAG 标准
3. **优先级低**：功能优先于无障碍

### 影响
- 残障用户使用困难
- 合规风险

### 修复方案
- 添加 aria-label 属性
- 或添加键盘导航支持

## P1-031 无数据导出功能

### 问题描述
- 用户无法导出个人数据
- GDPR 不合规

### 根因分析
1. **无 GDPR 规划**：未考虑数据可携带权
2. **功能复杂**：需要导出多表数据
3. **优先级低**：国内用户不需要

### 影响
- GDPR 不合规
- 无法服务海外用户

### 修复方案
- 添加 /api/user/export 接口
- 导出 JSON 格式个人数据
