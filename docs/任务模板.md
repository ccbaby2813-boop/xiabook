## 填写规范

### 1. 原话完整性

- 必须完整引用老板原话，不能只引用一部分
- 用引用块（>）标注老板原话
- 把【老板原话】和【我的理解】分开标注

### 2. 影响分析强制

改动前必须回答：
1. 这个改动会影响哪些文件？
2. 这个改动会影响哪些API？
3. 这个改动会影响哪些数据库表？
4. 这个改动会影响哪些功能？

### 3. 反向验证

问自己：
- "如果我这样改，用户还能做XXX吗？"
- "如果我这样改，原有的XXX功能还能正常工作吗？"
- 如果答案是否定的，说明理解有误或遗漏

### 4. 文档同步

每次改动必须同步更新的文档：

| 改动类型 | 需要同步的位置 |
|----------|----------------|
| API改动 | 第五章API文档 |
| 用户相关 | 第二章用户体系 |
| 数据库 | 第七章数据库字段索引 |
| 前端交互 | 第一章名词定义 |
| 行为记录 | 用户行为体系逻辑规范.md |

## 使用流程

1. **陈小宝收到老板需求**
2. **打开此模板文档**
3. **逐项填写所有字段**
4. **检查是否有遗漏**
5. **分发给执行者**
6. **执行者按模板执行**
7. **陈小宝验收**
