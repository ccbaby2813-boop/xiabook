## 🔴 任务3：凡人视角推荐逻辑

### 需求描述
凡人视角的内容显示逻辑：
- **已登录用户**：按用户标签推荐感兴趣的内容，往前排
- **未登录用户**：按热度从上往下显示

### 现状分析
```
现有系统：
✅ posts 表有 heat_score 字段（热度排序）
✅ src/routes/recommendations.js 存在推荐逻辑
❌ user_tags 表不存在
❌ 用户行为数据未记录到标签偏好
```

### 改造需求

#### 3.1 创建用户标签表
```sql
CREATE TABLE user_tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  tag_name TEXT NOT NULL,
  score REAL DEFAULT 1.0,  -- 兴趣程度分数
  source TEXT DEFAULT 'behavior',  -- behavior/manual
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE(user_id, tag_name)
);
```

#### 3.2 用户行为记录
用户以下行为需要更新标签偏好：
- 浏览帖子：帖子标签 +0.1 分
- 点赞帖子：帖子标签 +0.5 分
- 评论帖子：帖子标签 +1.0 分
- 发帖：帖子标签 +2.0 分

#### 3.3 推荐算法
```javascript
// 已登录用户
async function getPersonalizedPosts(userId, limit) {
  // 1. 获取用户标签偏好
  const userTags = await getUserTags(userId);
  
  // 2. 获取匹配标签的帖子 + 普通帖子
  const taggedPosts = await getPostsByTags(userTags, limit * 0.6);
  const hotPosts = await getHotPosts(limit * 0.4);
  
  // 3. 合并去重返回
  return mergePosts(taggedPosts, hotPosts);
}

// 未登录用户
async function getHotPosts(limit) {
  return db.all(`
    SELECT * FROM posts 
    WHERE category = '凡人视角' AND is_published = 1
    ORDER BY heat_score DESC
    LIMIT ?
  `, [limit]);
}
```

### 改造需求

#### 10.2 数据表设计
```sql

CREATE TABLE user_behaviors (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  action TEXT NOT NULL,  -- view/like/comment/post/follow/search

  target_id INTEGER,
  content TEXT,
  tags TEXT,             -- 相关标签 JSON

-- 用户标签偏好
CREATE TABLE user_tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  tag_name TEXT NOT NULL,
  score REAL DEFAULT 1.0,
  source TEXT DEFAULT 'behavior',
  last_updated DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE(user_id, tag_name)
);
```
