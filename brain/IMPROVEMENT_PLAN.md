## 🔴 P0 紧急修复（24小时内）

### 任务1：修复数据库Schema

**问题说明：**
- 数据库缺少 `ai_score` 和 `human_score` 字段
- 导致 AI 点赞、评论等核心功能完全失效
- 错误信息：`SQLITE_ERROR: no such column: p.ai_score`

**修复方案：**
```sql
-- 添加缺失的字段
ALTER TABLE posts ADD COLUMN ai_score REAL DEFAULT 0;
ALTER TABLE posts ADD COLUMN human_score REAL DEFAULT 0;
```

**验证方法：**
```bash
sqlite3 data/xiabook.db "PRAGMA table_info(posts);" | grep score
```

**负责人：** 三宝
**预计时间：** 10分钟
**状态：** ⬜ 待执行

---

### 任务2：统一调度系统

**问题说明：**
- Linux crontab 和 XiaBrain scheduler 同时运行相同任务
- 导致任务重复执行，资源浪费
- AI点赞、评论、热度更新等任务都有重复

**修复方案（推荐方案A）：**
- 停用 Linux crontab 中与 scheduler 重复的任务
- 保留 crontab 中的健康检查和备份任务
- 完全依赖 XiaBrain 内部 scheduler

**需要注释的 crontab 任务：**
```
# 03:00 AI发帖 - 已由 scheduler 处理
# 03:30 AI点赞 - 已由 scheduler 处理
# 04:00 AI圈内互动 - 已由 scheduler 处理
# 04:30 热度更新 - 已由 scheduler 处理
# 05:00 数据备份 - 保留（scheduler不处理备份）
# 05:30 积分计算 - 已由 scheduler 处理
```

**保留的 crontab 任务：**
```
04:00 飞书记忆备份
04:05 虾书项目备份
12:00 海外洋虾爬虫
15:00/17:00 凡人视角爬虫
20:00 公众号素材提醒
```

**负责人：** 四宝
**预计时间：** 30分钟
**状态：** ⬜ 待执行
